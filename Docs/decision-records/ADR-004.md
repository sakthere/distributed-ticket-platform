# ADR-004: JWT Authentication

Status: Accepted

## Context

The application should remain stateless and scalable.

## Decision

Authenticate users using JWT Bearer tokens.

## Alternatives

Sessions

Pros

Easy

Cons

Server-side storage

JWT

Pros

Stateless

Scalable

Works well with APIs

Cons

Cannot revoke easily

Requires refresh tokens

## Consequences

Authorization uses Claims.

No session storage required.
