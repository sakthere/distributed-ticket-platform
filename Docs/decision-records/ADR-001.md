# ADR-001: Adopt Vertical Slice Architecture

Status: Accepted

## Context

The project required an architecture that scales with features while remaining easy to understand during interviews and production development.

Traditional layered architecture groups files by technical concern (Controllers, Services, Repositories), often resulting in large service classes with unrelated responsibilities.

## Decision

Organize the application by feature.

Example:

Authentication/
    Register/
    Login/

Each feature owns:

- Command
- Handler
- Validator
- Result

## Alternatives Considered

### Layered Architecture

Pros

- Familiar
- Easy for small projects

Cons

- Large service classes
- Poor feature discoverability
- High coupling

### Vertical Slice (Chosen)

Pros

- Feature cohesion
- Easier maintenance
- Easier onboarding
- Naturally supports CQRS

Cons

- More folders
- Requires discipline

## Consequences

Positive

- Thin controllers
- Independent features
- Better scalability

Negative

- Initial learning curve
