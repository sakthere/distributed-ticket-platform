# ADR-003: Use BCrypt for Password Hashing

Status: Accepted

## Context

Passwords must never be stored in plaintext.

## Decision

Use BCrypt through an abstraction.

IPasswordHasher

↓

PasswordHasher

↓

BCrypt

## Alternatives

SHA256

Rejected

Reason

Fast hashes are unsuitable for passwords.

Plaintext

Rejected

Security risk.

## Consequences

Future hashing algorithm changes only require replacing the implementation.
